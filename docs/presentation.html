香港发卡方（Issuer）POC 演示

Mastercard Cloud Edge AWS Payment Cryptography ap-southeast-1

📋 演示背景

Mastercard 于 2025 年 6 月宣布推出 Cloud Edge 平台，与 AWS 合作让亚太地区银行和金融科技公司通过云端快速接入 Mastercard 支付网络，onboarding 时间缩短 4 倍。

本 POC 演示发卡方如何利用 AWS Payment Cryptography 替代传统物理 HSM，完成支付交易的密码学验证。

密钥别名用途算法

所有密钥存储在 AWS Payment Cryptography 托管 HSM 中，已通过 PCI PIN 和 PCI P2PE 认证。支持 TR-31 Key Block 格式与现有 HSM 互通。

🎬 演示场景

场景 1：电商交易 — CVV2 验证通过

持卡人在 HKTVmall 网购 HKD 299，输入正确的 CVV2。

# 正确 CVV2 → 批准
curl -s -X POST "$API_URL/authorize" \
  -H "Content-Type: application/json" \
  -H "x-api-key: $API_KEY" \
  -d '{
    "transaction_type": "ecommerce",
    "pan": "5425230000004415",
    "amount": 299, "currency": "HKD",
    "merchant": "HKTVmall",
    "expiry_date": "0127", "cvv2": "683"
  }'

预期结果：✅ 00 - Approved

要点：CVV2 由 AWS Payment Cryptography 使用 CVK 密钥实时验证，延迟 < 50ms。

场景 2：欺诈拦截 — CVV2 错误

盗卡者猜测 CVV2 进行交易，被即时拦截。

# 错误 CVV2 → 拒绝
curl -s -X POST "$API_URL/authorize" \
  -H "Content-Type: application/json" \
  -H "x-api-key: $API_KEY" \
  -d '{
    "transaction_type": "ecommerce",
    "pan": "5425230000004415",
    "amount": 1200, "currency": "HKD",
    "merchant": "Suspicious Store",
    "expiry_date": "0127", "cvv2": "999"
  }'

预期结果：❌ 05 - Do not honour

要点：错误 CVV2 被即时识别并拒绝，保护持卡人资金安全。

场景 3：ATM 取款 — PIN 验证

持卡人在铜锣湾 HSBC ATM 取款 HKD 5,000，输入正确 PIN。

# PIN 验证 → 批准
curl -s -X POST "$API_URL/authorize" \
  -H "Content-Type: application/json" \
  -H "x-api-key: $API_KEY" \
  -d '{
    "transaction_type": "atm",
    "pan": "5425230000004415",
    "amount": 5000, "currency": "HKD",
    "merchant": "HSBC ATM Causeway Bay",
    "encrypted_pin_block": "4AA2132737F32585",
    "pin_verification_value": "1064"
  }'

预期结果：✅ 00 - Approved

要点：PIN 明文从不出现在应用层，全程在 HSM 内处理，符合 PCI PIN 合规。

场景 4：未授权请求 — 安全防护

不带 API Key 的请求被 API Gateway 直接拒绝。

# 无 API Key → Forbidden
curl -s -X POST "$API_URL/authorize" \
  -H "Content-Type: application/json" \
  -d '{"transaction_type":"ecommerce","pan":"5425230000004415",...}'

预期结果：🚫 403 - Forbidden
